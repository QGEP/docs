2.2. Configuration du serveur de base de données

Si vous souhaitez accéder aux données depuis plusieurs postes de travail, vous pouvez installer la base de données sur un serveur de votre réseau. Pour un accès local uniquement, elle peut être installée sur le même poste.

2.2.1. Logiciels requis

2.2.1.1. Installation du serveur

  • Installer PostgreSQL (>=9.6)

    • Download page

  • Installer PostGIS (>=2.4)

        1. by using the (Application) Stack Builder shipped with the EnterpriseDB version of posgreSQL on Windows and macOS

Note

The following instructions are optional:

Pour un environnement de travail unique (pas de serveur en réseau), vous pouvez configurer votre base de données afin qu’elle accepte les connexions locales.

Ainsi votre base de données ne vous redemandera par le mot de passe si vous êtes connecté depuis votre poste de travail. Il est aussi possible de n’utiliser aucun mot de passe.

Cela peut être fait en éditant le fichier de configuration d’accès à la base de données pg_hba.conf. Pour cela changer l’option auth- de md5 à trust. Il faut ensuite recharger le service de base de données pour activer les changements.

Pour optimiser l’espace RAM utilisé etc. sur votre machine postgreSQL, il faut utiliser pgtune. Il est possible d’adapter votre fichier postgresql.conf file.
